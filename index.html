Promises ES6

Representam um valor que pode estar disponível agora, no futuro, ou nunca. Base do JavaScript assíncrono.

Criar & Encadear
const p = new Promise((resolve, reject) => {
  if (ok) resolve("sucesso");
  else   reject(new Error("falha"));
});

p
  .then(val => console.log(val))
  .catch(err => console.error(err))
  .finally(() => console.log("fim"));
Promise.all / race / allSettled
// Espera TODOS resolverem
Promise.all([p1, p2, p3])
  .then(([r1, r2, r3]) => { ... });

// O primeiro que resolver/rejeitar
Promise.race([p1, p2]);

// Todos, independente do resultado
Promise.allSettled([p1, p2])
  .then(results => results.forEach(r =>
    console.log(r.status) // "fulfilled" | "rejected"
  ));
Async / Await ES2017

Sintaxe que torna código assíncrono legível como síncrono. Sempre retorna uma Promise.

Padrão básico
async function buscarUsuario(id) {
  try {
    const res  = await fetch(`/api/users/${id}`);
    const data = await res.json();
    return data;
  } catch (err) {
    console.error("Erro:", err);
  }
}
Paralelo com await
async function carregarTudo() {
  // Dispara ambas ao mesmo tempo
  const [users, posts] = await Promise.all([
    fetch("/api/users").then(r => r.json()),
    fetch("/api/posts").then(r => r.json()),
  ]);
  return { users, posts };
}
Atenção: await só pode ser usado dentro de funções async. No top-level de módulos ES2022+ é permitido diretamente.

jQuery

Referência focada no uso real: seletores, DOM, eventos e — principalmente — requisições AJAX para o seu dia a dia no trabalho.

O que é jQuery?

jQuery é uma biblioteca JavaScript criada em 2006 que simplifica manipulação de DOM, eventos e AJAX com uma API consistente e cross-browser. Sua filosofia é "write less, do more".

Incluir no projeto
<!-- Via CDN (mais comum em projetos legados) -->
<script src="https://code.jquery.com/jquery-3.7.1.min.js"></script>

<!-- Via npm -->
// npm install jquery
import $ from "jquery";
DOM Ready — Ponto de entrada
// Aguarda o DOM estar pronto
$(function() {
  // todo o seu código jQuery aqui
  console.log("DOM pronto!");
});

// Equivalente mais verboso:
$(document).ready(function() { ... });
O símbolo $ é apenas um alias para a função jQuery(). Você pode usar jQuery(...) no lugar de $(...) em qualquer chamada.

Delegação de Eventos com .on()

Essencial para elementos criados dinamicamente (via AJAX, por exemplo). O evento é registrado no pai e filtrado pelo seletor filho.

// Sintaxe de delegação: $(pai).on(evento, seletorFilho, handler)

// Sem delegação — NÃO captura elementos futuros:
$(".item").on("click", function() { ... }); // ✗

// COM delegação — captura elementos futuros:
$("#lista").on("click", ".item", function() {
  console.log("item:", $(this).text());
});

// Delegação no document (mais ampla)
$(document).on("click", ".btn-excluir", function() {
  $(this).closest(".card").remove();
});
Dica prática: Sempre que adicionar elementos via AJAX e quiser ouvi-los, use delegação. É um dos padrões mais usados no dia a dia com jQuery.
$.ajax() — Requisições Completas AJAX

O coração do jQuery para comunicação com o servidor. Altamente configurável.

  },
  error: function(xhr, status, err) {

$.ajax({
  url: "/api/usuarios",
  method: "POST",

$.get() e $.post() — Atalhos AJAX

Para casos simples sem configuração avançada.

Atalho para GET que parse automaticamente o retorno como JSON.

  resultados.forEach(function(item) {

$.ajaxSetup() — Configuração Global AJAX

Define padrões para todas as requisições AJAX da página. Ideal para configurar headers de autenticação uma única vez.

// Colocar no início da aplicação (DOM Ready)
$.ajaxSetup({
  headers: {
    "Authorization": "Bearer " + localStorage.getItem("token"),
    "X-Requested-With": "XMLHttpRequest"
  },
  dataType: "json",
  contentType: "application/json",
  timeout: 15000,
  error: function(xhr) {
    // Handler global de erros
    if (xhr.status === 401) window.location = "/login";
  }
});

// Agora toda chamada herda essas configs
$.get("/api/me", function(data) {
  // Já usa o header Authorization automaticamente
});
Dica: Use $.ajaxSetup() no arquivo principal e evite repetir headers em cada chamada. É uma prática muito comum em sistemas legados com jQuery.
Deferred & Promise-like AJAX

As funções AJAX do jQuery retornam um objeto jqXHR que implementa a interface Deferred — parecida com Promises modernas.

.done() / .fail() / .always()
const req = $.ajax({ url: "/api/data" });

req
  .done(function(data) {
    console.log("sucesso", data);
  })
  .fail(function(xhr, status, err) {
    console.error("falhou", err);
  })
  .always(function() {
    $("#spinner").hide();
  });
$.when() — Paralelo
// Equivalente ao Promise.all
const r1 = $.get("/api/users");
const r2 = $.get("/api/posts");

$.when(r1, r2).done(function(res1, res2) {
  const users = res1[0]; // res1 = [data, status, xhr]
  const posts = res2[0];
  console.log(users, posts);
});

Criando um Plugin jQuery

Extender o jQuery com métodos customizados via $.fn.

// Padrão de plugin jQuery
(function($) {

  $.fn.destacar = function(cor) {
    cor = cor || "yellow";
    return this.each(function() {   // return this para encadeamento
      $(this).css("background", cor);
    });
  };

}($));

// Uso
$("p").destacar("#ffffaa").addClass("ativo"); // encadeável

Para o dia a dia, os métodos que você mais vai usar são: $(seletor) para selecionar, .on() para eventos, $.ajax() / $.get() / $.post() para requisições, .val() / .text() / .html() para ler/escrever dados, e .done() / .fail() para tratar respostas.
